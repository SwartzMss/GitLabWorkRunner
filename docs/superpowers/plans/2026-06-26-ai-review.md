# AI Review Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add native OpenAI-compatible AI review with automatic and manual GitLab MR triggers.

**Architecture:** Add AI review configuration to `rules.rs`, isolate provider request/response logic in a new `ai_review.rs`, and call it from `ReviewService` alongside existing line rules and script tasks. AI responses are converted into existing `Finding` values so comment grouping and GitLab publishing stay unchanged.

**Tech Stack:** Rust 2021, tokio, reqwest, serde, serde_json, globset, existing GitLab review pipeline.

---

## File Structure

- Modify `src/lib.rs` to export the new `ai_review` module.
- Create `src/ai_review.rs` for prompt construction, OpenAI-compatible API calls, JSON parsing, and added-line filtering.
- Modify `src/rules.rs` to parse and select `[[ai_reviews]]`.
- Modify `src/review.rs` to trigger AI reviews automatically and manually.
- Modify `tests/e2e_review.rs` to add a mock AI provider e2e test.
- Modify `README.md`, `README.en.md`, `rules.example.toml`, and `config.example.toml` documentation only after code behavior is green.

#[test]
fn parses_ai_review_defaults_and_selects_auto_reviews() {
    let rules = Ruleset::from_toml(
        r#"
[[ai_reviews]]
id = "ai-review"
title = "AI Review"
provider = "openai-compatible"
base_url = "https://api.openai.com/v1"
api_key_env = "OPENAI_API_KEY"

when_changed = ["src/**"]
"#,

    assert_eq!(reviews.len(), 1);
    assert_eq!(reviews[0].id, "ai-review");
    assert_eq!(reviews[0].trigger, AiReviewTrigger::AutoAndManual);
    assert_eq!(reviews[0].timeout_seconds, 60);
    assert_eq!(reviews[0].max_diff_bytes, 60000);
}
